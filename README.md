## What it does

Indexes 6,936 chunks from the Pipecat docs and exposes them through 4 MCP tools. Claude can answer Pipecat questions — configuration, code examples, architecture concepts, provider comparisons — with source-cited results pulled directly from the official docs.

**Retrieval pipeline:** BM25 (keyword) + FAISS dense vectors → Reciprocal Rank Fusion → Cross-encoder reranker

| Metric | Value |
|---|---|
| Pages indexed | 317 |
| Chunks indexed | 6,936 |
| Mean Reciprocal Rank | 0.788 |
| Recall@5 | 93% |
| Avg query latency | 36ms |

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/your-username/pipecat-docs-mcp.git
cd pipecat-docs-mcp
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Build the search index

The scraped `data/chunks.jsonl` is included. This step builds the BM25 and FAISS indexes (~2 min, writes ~30MB to `data/`):

```bash
python indexer.py
```

### 3. Connect to Claude Desktop

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "pipecat-docs": {
      "command": "/absolute/path/to/.venv/bin/python3",
      "args": ["/absolute/path/to/pipecat-docs-mcp/server.py"]
    }
  }
}
```

Use the Python binary from your virtual environment. Restart Claude Desktop after saving.

### 4. Connect to Cursor (HTTP)

```bash
fastmcp run server.py:mcp --transport http --port 8000
```

Then point your MCP client at `http://localhost:8000/mcp`.

## Project structure

```
pipecat-docs-mcp/
├── server.py              # FastMCP server — 4 MCP tools
├── retrieval.py           # Hybrid search (BM25 + FAISS + RRF + reranker)
├── indexer.py             # Builds BM25, FAISS, and metadata indexes
├── config.py              # Paths and model config
├── test_server.py         # In-process tests for all 4 tools
├── test_retrieval.py      # Retrieval eval — MRR, Recall@5, latency
├── requirements.txt
├── pipecat-scraper/
│   ├── scraper.py         # Sitemap-driven scraper (317 pages, resumable)
│   ├── utils.py           # HTML parsing and chunking
│   ├── audit.py           # Per-page extraction audit
│   └── config.py
└── data/
    ├── chunks.jsonl        # 6,936 scraped chunks (included)
    ├── chunk_metadata.json # Title, section, URL, content type per chunk
    ├── bm25_index.pkl      # Generated by indexer.py (gitignored)
    ├── faiss_index.bin     # Generated by indexer.py (gitignored)
    └── embeddings.npy      # Generated by indexer.py (gitignored)
```

## Re-scraping the docs

If you want to refresh the data against a newer Pipecat release:

```bash
cd pipecat-scraper
pip install -r ../requirements.txt
python scraper.py      # scrapes docs.pipecat.ai → data/chunks.jsonl
cd ..
python indexer.py      # rebuilds indexes
```
